今日有感

发表于2026-06-29|更新于2026-06-29|生活随笔
|浏览量:

今天继续在学习的道路上探索，有一些新的体会和思考想记录下来。从逻辑学的初次接触，到数据结构的深入学习，再到对自己博客功能的规划，每一步都让我对技术有了新的认识。

📚 逻辑学初体验

最近开始尝试学习逻辑学，看的是台湾大学的课程。不得不说，老师讲得很有意思，但内容确实有难度——听着听着就感觉迷迷糊糊的。

逻辑学作为一门基础学科，对编程思维的培养应该很有帮助。虽然现在理解起来还有些吃力，但我相信坚持下去会逐渐开窍的。

💡 学习心得

有些知识不是一次就能完全理解的，重要的是保持学习的热情和耐心。

🔗 双向链表学习进展

数据结构的学习还在继续，目前双向链表还没有完全学完。

现在的状态是：看懂了，但自己写还是有点困难。

我打算先把所有的课程系统地学完，建立完整的理论框架，然后再去实操练习。虽然这可能不是最快的学习方式，但我觉得打好基础更重要。

目前的学习策略

  1. 先理论后实践 — 完整学习课程内容
  2. 理解原理 — 不仅要知道怎么做，还要知道为什么这么做
  3. 延迟实操 — 等理论知识扎实后再大量练习

也许这种方式不是最优解，但它适合我现在的学习节奏。

💬 静态网站的评论功能

之前我一直以为挂载到 GitHub 上的静态网站是不可以加评论功能的。

后来才知道，通过一些第三方服务（比如 Gitalk、Utterances、Valine 等），静态网站也能实现评论功能！等什么时候 Codex 能用了，我也想给自己的博客加上这个功能。

这让我意识到：很多时候限制我们的不是技术本身，而是我们对技术的认知。

🎯 一个小目标

“什么时候可以靠自己实现各种技术，不靠 AI 了？”

这是我今天问自己的一个问题。

AI 工具确实很强大，能帮我们快速解决问题，但我不想成为只会调用 API 的”工具人”。我希望有一天，我能够真正理解技术的底层原理，靠自己的能力去实现各种功能。

这可能需要很长时间，但这就是学习的意义所在——从依赖工具，到理解工具，再到创造工具。

今日总结

  • ✅ 接触了逻辑学，虽然有点难，但很有意思
  • ✅ 继续学习数据结构，双向链表理论部分接近完成
  • ✅ 了解到静态网站也能加评论功能
  • ✅ 明确了学习目标：不只是会用，更要真正理解

学习是一个持续的过程，每天进步一点点，积累起来就是质的飞跃。

加油！ 💪
